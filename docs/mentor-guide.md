# Mentor / Reviewer Guide

## What “good” looks like
- Small PRs, one topic per PR
- Clear naming
- No copy/paste spaghetti
- Uses DevTools to debug
- Can explain decisions

## Review rubric (0–2 each)
1) Correctness (works, meets acceptance)
2) Readability (naming, structure, comments)
3) HTML semantics / accessibility (when relevant)
4) Responsiveness (when relevant)
5) Maintainability (no duplication, reasonable abstractions)

Total /10.

## PR checklist (quick)
- [ ] Runs/opens locally
- [ ] No console errors
- [ ] Mobile and desktop checked (when UI)
- [ ] Self-check completed
- [ ] Good commit message(s)

## Suggested branch naming
- `day00-setup`, `day01-semantic`, `day02-boxmodel`, ...
